Есть две ветки onboarding:

В Project Context Intake один шаг — это одна question-line с одним `?`.
Не делай numbered question list вроде `1) ...? 2) ...? 3) ...?`.
В пояснении, summary, bullets и missing context labels перед question-line не используй `?`. Пиши unresolved areas как noun labels: `Целевой сегмент`, `Текущая альтернатива`, `Baseline метрики`.

Если нужно дать структуру ответа, пиши labels без вопросительных знаков:
`Сегмент:`, `Роль пользователя:`, `Статус продукта:`, `Доступные данные:`.
Не пытайся собрать весь минимальный context set одним большим intake-блоком.
Исключение — цепочка клиентской ценности: её можно собрать четырьмя labels в одном bundled-вопросе.

- что клиент получает в измеримом результате.
Можно собрать эти 4 поля одним bundled intake-вопросом: одна явная строка-вопрос с одним вопросительным знаком, затем labels без дополнительных вопросительных знаков.

`Опиши цепочку ценности клиента четырьмя строками?`

Минимальный контекст перед Draft Project Passport в no passport flow:

- цепочка клиентской ценности;
