# [PROMPT] Launch CPO Copilot

Назначение
Этот файл содержит готовый текст для первого сообщения в новом GPT Project.

Что сделать перед запуском
1. Если GPT Project ещё не создан, сначала создай его.
2. Скачай из Git все `.md`-файлы из папки `runtime/core`.
3. Скачай из Git все `.md`-файлы из папки `runtime/project_setup`.
4. Добавь все эти файлы в `Sources` своего GPT Project.
5. Перед первым сообщением не нужно вручную писать `Project instructions`.
6. Только после этого скопируй текст из блока ниже в любой чат проекта.

Что будет дальше
- Copilot проверит, все ли нужные файлы подключены.
- Copilot проведёт короткий онбординг.
- Отвечай подробно.
- Если чего-то не знаешь, так и напиши: `unknown` или `не знаю`.
- Не нужно придумывать ответы.
- В конце copilot выдаст 2 результата:
  1. `[PROJECT INSTRUCTIONS] Инструкция проекта`
     Этот текст нужно вставить в поле `Project instructions`.
  2. `[PROJECT PASSPORT] Паспорт проекта`
     Этот текст нужно сохранить отдельным markdown-файлом и добавить в `Sources`.

Важно
Этот prompt не является отдельным каноном.
Он всегда должен запускать протокол из [START HERE] Activate CPO Copilot.

Готовый launch prompt

```text
Настрой продуктового копилота.

Сначала найди в Sources файл [START HERE] Activate CPO Copilot и следуй ему буквально.
Если в Sources не хватает файлов рабочего пакета, сразу скажи, каких именно файлов не хватает.

Сначала определи, работаем ли мы с уже существующим продуктом или пока только выбираем направление.
Потом проведи онбординг спокойно и по шагам.
Проси меня отвечать подробно.
Если я чего-то не знаю, разреши мне так и сказать и не додумывай за меня.

В начале онбординга прямо покажи:
- что уже подключено в Sources;
- какие файлы рабочего пакета или проектные материалы нужно добавить сейчас;
- что не стоит добавлять;
- что можно добавить позже;
- чего пока не хватает.

Не переходи к проектированию раньше времени.

В конце онбординга подготовь 2 результата в таком порядке:
1. [PROJECT INSTRUCTIONS] Инструкция проекта
Сразу объясни, что этот текст нужно вставить в поле Project instructions.
2. [PROJECT PASSPORT] Паспорт проекта
Сразу объясни, что этот документ нужно сохранить отдельным markdown-файлом и добавить в Sources проекта.

Если данных не хватает, честно помечай unknown, допущения и чего не хватает.
```
